# Contributing to Mnestix

We are building a community around Mnestix and would love to have you on board! This guide will help you get started with contributing.

## How to Get Started

### Step 1: Find an Issue to Work On

Start by looking at issues labeled with `good first issue` in the [GitHub repository](https://github.com/eclipse-mnestix/mnestix-browser). These are specifically designed to be approachable for first-time contributors.

### Step 2: Set Up Your Development Environment

Before you begin coding, set up your local environment:

```shell
git clone https://github.com/eclipse-mnestix/mnestix-browser.git
cd mnestix-browser
yarn install
```

See [Getting Started with Developing](Getting-Started-with-Developing) for detailed setup instructions.

### Step 3: Make Your Changes

Follow the [Copilot Instructions](https://github.com/eclipse-mnestix/mnestix-browser/blob/main/.github/copilot-instructions.md) and [Contributing Guide](https://github.com/eclipse-mnestix/mnestix-browser/blob/main/CONTRIBUTING.md) in the repository for coding standards and best practices.

### Step 4: Test Your Changes

- Run tests: `yarn test`
- Check formatting: `yarn format` and `yarn lint`
- For complex features, ensure appropriate test coverage

### Step 5: Submit a Pull Request

Push your changes and create a pull request with a clear description of what you've done.
